Copper Syntax

Copper sticks to a very particular syntax pattern within your markup, and works very hard not to stray far from it. Just about everything you do in Copper (with the exception of manually managing stores) follows this basic pattern:

HTML 

                        
<tag cu-operation="source -> destination">

    Default content

</tag>

Binding data

Let's apply that to binding data to an element's property.

HTML 

            
<!-- Bind the value of store1 to the element's innerText attribute -->
<p cu-bind="store1->innerText">Default text</p>

Pretty straightforward, but if you were to code that up and visit your site, the value of <p>'s innerText attribute would be "Default text." The reason should be obvious — store1 was never initialized!

Let's move on to the next section, where we can load store1 with a value using cu-sync. Don't worry, code purists — you can initialize and manipulate store values in your scripts, too. We'll get to that soon!
