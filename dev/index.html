What is Copper?

Some words about Copper.

Installation

Copper was made to be flexible, so it can be used all on its own or alongside other libraries and frameworks in a JS build system.

TypeScript 

            
// You can also import Copper as types-provided module. `npm install @jwrunge/copper`, then:
import {Cu} from '@jwrunge/copper';

// OR for CommonJS:
const Cu = require('@jwrunge/copper');
HTML 

            
<!-- Download the script and include it in your project: -->
<script src="./scripts/copper.js"></script>

<!-- You can also download the 10KB "slim" version, which does not bundle JSDoc types: -->
<script src="./scripts/copper.slim.js"></script>

Types and intellisense:

When you include Copper in your project, you get a full set of types for the library, even if you aren't using TypeScript (though you do need to set up your editor to type-check your code).

As long as you are working within a `.js` or `.ts` file (and not within a `<script>` tag in an HTML file), you can use TypeScript's triple-slash `reference` directive to point to Copper types. This is extra useful because it means that you can use `slim.copper.js` in your code and type check against `copper.js`, saving your users' bandwidth! For this to work, you do need to have the fully-typed `copper.js` file accessible locally, as types cannot be resolved over a network.

JavaScript 


// Include types from copper.js
/// <reference path="./scripts/copper.js" />
